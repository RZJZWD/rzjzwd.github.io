打开UVC支持和相关工具

操作内核配置
在SDK目录下使用./build.sh kconfig进入内核编译，然后
Device Driver–>Multimedia support–>Media drivers–>按Y选择Media USB Adapters并进入–>选择USB Video Class UVC input events device support

buildroot配置用户空间工具
在SDK目录下使用./build.sh bconfig进入buildroot编译，然后

v4l2

Target packages–>Libraries–>Hardware handling–>选择libv4l以及下面的v4l-utils tools和另一个插件BR2_PACKAGE_LIBV4L_BUILTIN_PLUGINS

gstreamer

Target packages–>Audio and video applications–>选择ygstreamer 1.x–>开启以下的插件

buildroot编译和网络问题

使用./build.sh bconfig配置镜像站，具体镜像站网上查
export RK_NETWORK_CHECK=n来关闭网络检查

如果还是下不动，就更新CA证书

如果libv4l报错

这是_TIME_BITS=64编译错误 编译器要求定义_TIME_BITS=64时必须也要定义_FILE_OFFSET_BITS=64
经过我不断的尝试，包括但不限于打补丁，根据报错信息改源码等方法，即使是手动定义_FILE_OFFSET_BITS=64也会有64位文件操作函数的问题

最终解决办法是对这个库取消_TIME_BITS宏定义
在/home/forlinx/work/OK3506_Linux_Source/buildroot/package/libv4l/下创建local.mk对原始的mk文件进行追加设置，
写入LIBV4L_CONF_ENV = CFLAGS="$(TARGET_CFLAGS) -U_TIME_BITS"
类似的问题

如果后面再遇到_TIME_BITS=64编译错误这个问题，直接关了就行
