vs2026配置dotnet8安卓跨平台开发

发表于 2026-04-25 更新于 2026-04-30 分类于 Windows ， .NET
vs2026配置低版本Android跨平台开发环境

前言

在使用vs2026 .NET 10 avalonia 12开发跨平台拉流应用时，使用LibMPVSharp.Avalonia，但是它的运行时文件要求avalonia版本不高于11.2.3，部署时表现为ui渲染出来但是程序崩溃

解决方法为降级

  • 将avalonia降级到11.2.3
  • avalonia 11.2.3 要求.net8，所以要一起降级

开始降级

降级net8

先创建.net8工程
然后在工程目录下创建global.json文件，写入

这里强制.NET版本，并且设置为向前滚动到最新，而不会使用.net9/10
注意 这个文件放在工程根目录而不是共享解决方案

重启工程后在工程根目录下打开终端输入dotnet --version即可看到改变

降级avalonia版本

在工程根目录下找到Directory.Packages.props，这个是整个工程控制包的文件，打开编辑，将avalonia的12.0.1->11.2.3

重新下载Android工作负载

如果不重下属于.net8的Android工作负载，会出现报错

解决方案：包 Avalonia.Android 11.x.x 与 net8.0-android21.0 (.NETCoreApp,Version=v8.0) 不兼容。 包 Avalonia.Android11.x.x 支持: net8.0-android34.0 (.NETCoreApp,Version=v8.0)

接下里输入dotnet workload install android下载工作负载，这里你能看到是.net8的内容
下载完后重启工程，一路确定就行

之后就能在myapp-android中调整api版本了！

补充

avalonia11.2.3的安卓启动入口点程序稍有变化

如果运行时出现xxx找不到，注意你加入的其他包是否有对avalonia的依赖，这会导致引入高版本avalonia
